                              PROBLEM STATEMENT
================================================================================

User reported browser security error:
  "Unsafe attempt to load URL http://localhost:3000/ from frame with URL
   chrome-error://chromewebdata/. Domains, protocols and ports must match."

Root Cause Analysis:
  1. Backend CORS configuration too permissive (wildcard origins)
  2. Frontend axios missing credentials configuration
  3. Improper cross-origin request handling

================================================================================
                            SOLUTION IMPLEMENTED
================================================================================

CHANGE 1: Backend CORS Configuration
File: backend/src/main.py (lines 60-75)

BEFORE:
  app.add_middleware(
      CORSMiddleware,
      allow_origins=["*"],
      allow_credentials=True,
      allow_methods=["*"],
      allow_headers=["*"],
  )

AFTER:
  cors_origins = [
      "http://localhost:3000",
      "http://127.0.0.1:3000",
      "localhost:3000",
  ]

  app.add_middleware(
      CORSMiddleware,
      allow_origins=cors_origins,
      allow_credentials=True,
      allow_methods=["GET", "POST", "PUT", "DELETE", "PATCH", "OPTIONS"],
      allow_headers=["*"],
      expose_headers=["*"],
      max_age=3600,
  )

Impact:
  ✅ Specific origins instead of wildcard
  ✅ Explicit HTTP methods
  ✅ Credentials enabled
  ✅ Headers exposed to frontend
  ✅ Preflight requests cached (1 hour)

================================================================================

CHANGE 2: Frontend Axios Configuration
File: frontend/lib/axios.ts (line 14)

BEFORE:
  const axiosInstance = axios.create({
    baseURL: process.env.NEXT_PUBLIC_API_URL || 'http://localhost:8000',
    headers: {
      'Content-Type': 'application/json',
    },
  })

AFTER:
  const axiosInstance = axios.create({
    baseURL: process.env.NEXT_PUBLIC_API_URL || 'http://localhost:8000',
    headers: {
      'Content-Type': 'application/json',
    },
    withCredentials: true,
  })

Impact:
  ✅ Credentials sent with cross-origin requests
  ✅ Cookies and auth headers included
  ✅ Proper CORS handling
