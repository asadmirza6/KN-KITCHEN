# 🚀 KN Kitchen - Deployment Guide

## Overview
KN Kitchen has two parts:
- **Frontend (Next.js)** → Deploy on Vercel
- **Backend (FastAPI)** → Deploy on Railway/Render/Heroku

---

## 📦 Frontend Deployment (Vercel)

### Option 1: Using Vercel Dashboard (Recommended)

1. **Go to Vercel:**
   - Visit: https://vercel.com
   - Login with GitHub

2. **Import Project:**
   - Click "Add New..." → "Project"
   - Select: `asadmirza6/KN-KITCHEN`
   - Click "Import"

3. **Configure Build Settings:**
   ```
   Framework Preset: Next.js
   Root Directory: frontend
   Build Command: npm run build
   Output Directory: .next
   Install Command: npm install
   ```

4. **Environment Variables:**
   Add these in Vercel Dashboard:
   ```
   NEXT_PUBLIC_API_URL=https://your-backend-url.railway.app
   ```

5. **Deploy:**
   - Click "Deploy"
   - Wait for deployment to complete
   - Your frontend will be live at: `https://kn-kitchen.vercel.app`

### Option 2: Using Vercel CLI

```bash
# Install Vercel CLI
npm i -g vercel

# Navigate to frontend
cd frontend

# Deploy
vercel

# Follow prompts:
# - Set up and deploy? Yes
# - Which scope? Your account
# - Link to existing project? No
# - Project name? kn-kitchen
# - Directory? ./
# - Override settings? No

# Production deployment
vercel --prod
```

## 🖥️ Backend Deployment (Railway - Recommended)

### Why Railway?
- ✅ Easy PostgreSQL database
- ✅ Free tier available
- ✅ Automatic deployments from GitHub
- ✅ Environment variables support

### Steps:

1. **Go to Railway:**
   - Visit: https://railway.app
   - Login with GitHub

2. **Create New Project:**
   - Click "New Project"
   - Select "Deploy from GitHub repo"
   - Choose: `asadmirza6/KN-KITCHEN`

3. **Configure Service:**
   ```
   Root Directory: backend
   Build Command: pip install -r requirements.txt
   Start Command: uvicorn src.main:app --host 0.0.0.0 --port $PORT
   ```

4. **Add PostgreSQL Database:**
   - Click "New" → "Database" → "Add PostgreSQL"
   - Railway will auto-create DATABASE_URL

5. **Environment Variables:**
   In Railway Dashboard, add:
   ```
   DATABASE_URL=<auto-generated-by-railway>
   BETTER_AUTH_SECRET=your-secret-key-here-min-32-chars
   JWT_ALGORITHM=HS256
   JWT_EXPIRE_DAYS=7
   CORS_ORIGINS=["https://kn-kitchen.vercel.app"]
   CLOUDINARY_CLOUD_NAME=your-cloudinary-name
   CLOUDINARY_API_KEY=your-cloudinary-key
   CLOUDINARY_API_SECRET=your-cloudinary-secret
   ```

6. **Run Database Migrations:**
   In Railway console:
   ```bash
   alembic upgrade head
   ```

7. **Get Backend URL:**
   - Railway will give you a URL like: `https://kn-kitchen-backend.railway.app`
   - Copy this URL

8. **Update Vercel Environment Variable:**
   - Go back to Vercel Dashboard
   - Update `NEXT_PUBLIC_API_URL` with Railway backend URL
   - Redeploy frontend

## 🔄 Alternative: Backend on Render.com

### Steps:

1. **Go to Render:**
   - Visit: https://render.com
   - Login with GitHub

2. **Create Web Service:**
   - New → Web Service
   - Connect GitHub: `asadmirza6/KN-KITCHEN`

3. **Configure:**
   ```
   Name: kn-kitchen-backend
   Root Directory: backend
   Environment: Python 3
   Build Command: pip install -r requirements.txt
   Start Command: uvicorn src.main:app --host 0.0.0.0 --port $PORT
   ```

4. **Add PostgreSQL:**
   - New → PostgreSQL
   - Connect to web service

5. **Environment Variables:**
   Same as Railway (see above)

6. **Deploy:**
   - Click "Create Web Service"
   - Wait for deployment
   - Get backend URL and update Vercel

## 🔧 Quick Fix for Vercel 404 Error

If you're getting 404 error on Vercel:

### Method 1: Set Root Directory in Vercel Dashboard
1. Go to Project Settings
2. General → Root Directory
3. Set to: `frontend`
4. Redeploy

### Method 2: Use vercel.json (Already Added)
The `vercel.json` file in root tells Vercel where frontend is.

### Method 3: Deploy from Frontend Folder
```bash
cd frontend
vercel --prod
```

## 🌍 Environment Variables Reference

### Frontend (.env.local)
```env
NEXT_PUBLIC_API_URL=https://your-backend.railway.app
```

### Backend (.env)
```env
# Database (Auto-generated by Railway/Render)
DATABASE_URL=postgresql://user:password@host:5432/dbname

# Auth
BETTER_AUTH_SECRET=generate-random-32-char-string
JWT_ALGORITHM=HS256
JWT_EXPIRE_DAYS=7

# CORS (Add your Vercel URL)
CORS_ORIGINS=["https://kn-kitchen.vercel.app","http://localhost:3000"]

# Cloudinary
CLOUDINARY_CLOUD_NAME=your-cloud-name
CLOUDINARY_API_KEY=your-api-key
CLOUDINARY_API_SECRET=your-api-secret
```

## 🐛 Troubleshooting

### Frontend Issues:

**Problem:** 404 on Vercel
**Solution:** Set Root Directory to `frontend` in project settings

**Problem:** API calls failing
**Solution:** Check `NEXT_PUBLIC_API_URL` environment variable

**Problem:** Build failing
**Solution:** Check Node.js version (should be 18.x or higher)

### Backend Issues:

**Problem:** Database connection error
**Solution:** Verify DATABASE_URL is set correctly

**Problem:** CORS errors
**Solution:** Add frontend URL to CORS_ORIGINS

**Problem:** Migrations not applied
**Solution:** Run `alembic upgrade head` in Railway/Render console
