================================================================================
PROBLEM FIXED
================================================================================

Manual items were calculating correctly on the frontend but NOT being saved to
the database. They were missing from:
- Order list
- Order details view  
- PDF invoices

ROOT CAUSE:
- Order model lacked manual_items column
- create_order() wasn't explicitly saving manual_items
- API responses didn't include manual_items
- PDF generation didn't iterate through manual_items
- Frontend didn't display manual_items in order details

================================================================================
SOLUTION IMPLEMENTED
================================================================================

1. DATABASE SCHEMA (backend/src/models/order.py)
   - Added manual_items column (JSON type)
   - Stores array of manual item objects
   - Structure: [{"name": "...", "quantity_kg": X, "price_per_kg": Y, "subtotal": Z}]

2. API LOGIC (backend/src/api/orders.py)
   - create_order(): Processes and saves manual_items
   - update_order(): Handles manual_items independently
   - All responses: Include manual_items field
   - PDF generation: Includes manual_items in invoice

3. FRONTEND (frontend/app/admin/orders/page.tsx)
   - Order interface: Added manual_items field
   - Order details modal: Displays manual_items with separator
   - Shows manual items in same table as menu items

4. DATABASE MIGRATION
   - Generated: fec3103de63a_add_manual_items_column_to_orders_table.py
   - Applied: Migration successfully run
   - Result: manual_items column now exists in PostgreSQL
